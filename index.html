Role-isolated multi-agent TDD pipeline

shipped
A 12-phase pipeline with deliberate context isolation between agents.

Splits feature and bug work across specialized agents — research, test-writing, implementation, review, spec-compliance, QA generation — each seeing only what it needs. Implementation agents are kept blind to acceptance criteria (they see only failing tests) to prevent "testing to the answer," while cheaper models handle review and stronger models handle code generation.
